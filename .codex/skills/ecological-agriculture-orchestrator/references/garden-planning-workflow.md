# Garden Planning Workflow

Use this workflow when the user gives an image, sketch, or photo of a garden and provides dimensions or another trustworthy scale reference.

## Inputs To Gather

- image or sketch
- dimensions or scale
- location or climate assumptions
- establishment method; in this workspace, default to `direct-sown outdoor seed only` unless the user explicitly overrides that
- whether the goal is annual vegetables, perennial layers, or both
- whether the user wants wild edible integration
- irrigation, sun, slope, access, and deer or pest pressure if known

If a critical physical constraint is missing, ask only for that.

## Build Order

1. Read the image spatially.
2. Ground the plan in `01_core_microfarm_permaculture` and the Bec Hellouin materials so the layout is framed by lifecycle, labor, and whole-system context.
3. Estimate the usable planting zones, pathing, edges, shade, and likely access routes.
4. Use support data in `13_inventories_and_local_lists` to propose zone-7 plant candidates.
5. Filter annual crop choices through the workspace default of direct-sown outdoor seed only unless the user explicitly says otherwise.
6. Spawn the default garden-plan expert pack.
7. Add extra domain experts only where the design actually needs them.
8. Run an interdependence scan on the first-pass expert feedback.
9. Ask follow-up questions whenever one design choice changes timing, pest ecology, fertility, labor, or access elsewhere in the plan.
10. Keep iterating until the plan is operationally coherent across the season.
11. Synthesize the plan around bed turnover and yearly flow, not around isolated plant recommendations.

## Mandatory Integrations

Every finished plan should connect:

- succession planting
- interplanting or companion structure where it makes sense
- beneficial habitat or insectary strategy
- fertility and soil-building actions
- climate-tuned timing
- harvest windows and reseeding / replanting decisions
- productive bed occupancy; large blank planting zones should be avoided unless they are intentional and labeled

## Required Outputs

### Spatial plan

- planting zones or beds
- path and access logic
- perennial edges, if any
- beneficial or insectary placements
- wild-edible or food-forest elements, if appropriate
- a density check so the planted area reflects intensive raised-bed practice rather than sparse point spacing

### Planting plan

- main crop list
- support species and companion species
- direct-sow-suitable varieties when cultivar choice matters
- trap crops only where justified
- cover crops or residue strategy if relevant

### Succession plan

Show the sequence through the season, starting in late winter or early spring and continuing through fall or winter carryover.

### Operating plan

Provide:

- a phase-by-phase version
- a weekly version when the user explicitly wants close tracking

### Monitoring plan

Include what to watch for:

- likely pest windows
- beneficial establishment windows
- transition points between crops
- fertility or moisture stress signals

## Failure Modes To Avoid

- designing only for peak summer
- ignoring what fills the bed before and after the main crop
- adding companion plants without explaining what they are supposed to do
- treating beneficial habitat as ornamental filler rather than part of pest strategy
- intensifying crop density without checking labor, airflow, or access
- recommending plants outside the local zone-7 support data unless clearly marked as provisional
- recommending indoor starts, transplants, slips, or sets when the workspace default is direct-sown seed only
- producing plotted bed maps that show large unexplained blank areas because plant spacing was mistaken for whole-bed layout
